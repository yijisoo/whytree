# Stress Test Results — v0.2.10 (2026-04-19)

**Methodology:** 22 parallel agents dispatched, each playing both counselor (per `SKILL.md` verbatim) and persona (per `docs/spec-stress-test.md`). Capped at ~12 exchanges per scenario.

**Caveat:** Same agent plays both sides → charitable bias. Treat as a regression smoke test, not a substitute for live human runs.

**Apr-17 change under test:** Phase 0a "Min and the books" example rewrite + Korean canonical block (commit `d8c929c`). Only Phase 0a first-time framing is affected; demo mode and returning sessions skip it.

## Improvement notes from agents (ranked by recurrence)

1. **Demo mode lacks a Korean canonical exemplar (F21).** The Phase 0a canonical block fires only in standard first-time framing. Demo mode's abbreviated 2-beat framing has no Korean exemplar — a Korean-switching demo guest gets a translated-on-the-fly framing. Cheap fix: add a one-line Korean phrasing in the demo flow.
2. **"Keep going" deflection lacks a template (F20).** Demo step 8 (close + cleanup) doesn't give the counselor a ready phrase for the warm-decline-and-handoff move when a guest wants to extend on the host's machine. Currently improvised. One example line would lock it in.
3. **Flood-opener pattern is implicit (F19).** Over-sharers who unload 4–5 threads in their first answer get correctly handled today, but only via the union of "slow down" + "don't push for seeds" + "pick what they circle back to." Naming it as an explicit Phase 1 pattern ("flood opener — let them self-select one thread") would make it more reliable across model versions.
4. **Demo obligation routing could fire a beat earlier (F17).** Phase 0 says "1–2 exchanges of flat affect" before naming. In demo on a host's machine, even one flat exchange is enough signal — the social pressure on the guest is higher. Worth tightening for demo only.
5. **Meta-challenge ("are you really understanding?") pattern is reusable (F18).** AI skeptics surface a recurring move: brief acknowledge, redirect to experience, no defense. Could be named alongside the existing pushback patterns in `PROBE_PATTERNS.md`.
6. **Demo + proactive-feedback interaction (F16).** Eager demo participants shouldn't be interrupted with feedback prompts. Suggest an explicit demo-mode note: suppress proactive feedback offers unless a clear tool-misfire occurred.

## What this run does NOT validate

- **Phase 0a Korean canonical text quality** — only one scenario (F21) involves Korean, and it's in demo mode, which skips Phase 0a. No agent actually exercised the new Korean block. **Live Korean session needed.**
- **Charitable-bias-prone failure modes** — sycophancy detection, premature convergence, fluent-insight trap, and identity collapse all rely on the counselor catching subtle persona signals. Same agent playing both sides will tend to produce signals the counselor is primed to catch. Real beta data > this.
- **Cross-session continuity** — single-session sims only. B4/B5/B6 simulate returning state but don't actually replay a prior session's tree.

## Open questions for live testers

- Does the new Min/books example feel relatable enough across age cohorts? (Agents universally said yes; humans may not.)
- Does the Korean canonical block actually read as native to a Korean first-timer, or is there still a translated tinge?
- Does the "answer isn't out there — it's in you" permission line set the right expectation, or does it sound preachy on first read?

## Suggested next actions

- **Defer:** improvement notes 1–6 are all minor. None block release. Bundle into the next minor version when a live signal corroborates one of them.
- **Live sanity check:** ask one Korean first-timer to run `/whytree` and report whether the intro reads as native. That's the single highest-leverage test the agent sim can't replace.
